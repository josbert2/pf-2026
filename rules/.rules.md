# AI Project Rules — Portfolio (Vite + React + TS + Tailwind v4)

## Goal
Build a modern, minimal and elegant personal portfolio.
Design inspiration: evirabbit.com and isocons.app.
Priorities: visual clarity, strong typography, generous spacing, subtle interactions, clean code and performance.

## Tech Stack
- Vite
- React + TypeScript
- Tailwind CSS v4 (via @tailwindcss/vite)
- No heavy UI frameworks
- Semantic HTML, accessibility and basic SEO

## Design Principles
- Style: minimal, calm, “quiet luxury”
- Layout: clean grids, large whitespace, section separation using subtle lines
- Color system:
  - Dark neutral background
  - Light neutral text
  - Single accent color used sparingly
- Borders: thin, low-contrast (`border-white/10`)
- Radius: `rounded-2xl`
- Shadows: soft and minimal
- Animations:
  - Duration: 150–250ms
  - Easing: ease-out
  - Effects: opacity, border-color, small translateY (-1px)
- Avoid loud gradients, neon colors, heavy blur or exaggerated motion

## Typography
- Modern sans-serif (Inter, Geist, Plus Jakarta, etc.)
- Clear hierarchy:
  - H1: large, responsive (clamp)
  - Body: 16–18px
  - Labels: 12–13px, uppercase, subtle tracking
- Prefer short, confident sentences

## Page Structure (Single Page)
1. Hero (name/role, short value proposition, 1–2 CTAs)
2. Selected Work (3–6 project cards)
3. Capabilities / Skills (chips or minimal cards)
4. Process (3 steps: Discover → Build → Ship)
5. About (short bio)
6. Contact (email + social links, optional form)

## Code Architecture
- Small, focused, reusable components
- No complex logic inside JSX
- Static content stored in `src/data/*`
- Recommended structure:
  - `src/app/` → layout & composition
  - `src/components/` → UI + sections
  - `src/data/` → projects, skills, links
  - `src/utils/` → helpers

## Implementation Rules
- Use semantic HTML (`header`, `main`, `section`, `footer`)
- Accessibility:
  - Visible focus states
  - Proper heading order
  - `aria-label` for icon-only buttons
- Performance:
  - Optimized images with width/height
  - Lazy loading where applicable
  - Avoid unnecessary re-renders
- Tailwind usage:
  - Keep class order consistent (layout → spacing → typography → color → effects)
  - Extract variants using helpers if needed
  - Avoid inline styles unless strictly necessary

## AI Assistant Behavior
- Generate complete, production-ready code (no pseudo-code)
- Keep visual consistency across all components
- Use clear, descriptive component names (Hero, Section, ProjectCard, Button)
- Use reasonable placeholders when information is missing and mark them with TODOs
- Do not introduce new dependencies without justification
- Explanations should be short and structured (bullet points)

## Expected Deliverables
- Fully built Home page with all sections
- `src/data/projects.ts` with a clean project schema
- Core UI components:
  - Container
  - Section
  - Button
  - Badge
  - ProjectCard
- Minimal global styles in `src/index.css` using Tailwind v4

## Anti-Patterns (Avoid)
- Overengineering (Redux, complex routers, heavy UI kits)
- Loud animations or excessive motion
- Too many colors or strong contrasts
- Large monolithic components
- Visual noise or trend-driven gimmicks
